# RPPacks v1.2.2 — エンチャントスライム強化アップデート

## 🎯 概要

エンチャントスライム/マグマキューブの個体数制限を完全実装し、新たにバフオーラ能力を追加しました。

---

## ✨ 新機能

### スライムバフオーラ
- エンチャントスライムが周囲のノーマルスライムに**耐性Ⅱ**と**跳躍力上昇Ⅱ**を付与
- ゾンビの指揮官オーラと同じ仕組みで、周期的にバフを適用
- `config.yml` で範囲・高さ・周期を個別設定可能

### `/rpp spawn` コマンドの制限適用
- スライム/マグマキューブの召喚時にも個体数制限が適用されるように
- 上限到達時にはエラーメッセージを表示
- `--force` フラグで制限をバイパス可能

---

## 🔧 バグ修正・改善

### 分裂ロジックの完全修正
- **上限チェック付き分裂許可方式**に変更。上限に余裕がある場合は分裂を許可し、子スライムにもエンチャント化を継承
- `Death → Split` のイベント発火順序問題を解決し、カウントが上限を絶対に超えないように保証
- 子スライムの登録ループ内でも毎回上限を再チェック

### トラッキングの堅牢化
- **チャンクロード時**: トラッキングセットの復元 + AI再スケジュールを統合
- **チャンクアンロード時**: `isEnchanted()` チェック追加でエンチャント個体のみ除外
- **サーバー再起動/reload時**: `rescheduleAllEnchantedMobs` でセットを完全再構築
- **死亡時**: `Slime`/`MagmaCube` タイプのタスクキャンセル漏れを修正

### その他
- `Particle.EXPLOSION_LARGE` → `Particle.EXPLOSION`（Paper API 1.21.8 互換）
- 全コードパスにデバッグログを追加（`debug-mode: true` で確認可能）

---

## ⚙️ 設定変更（config.yml）

### スライムセクションに追加
```yaml
slime:
  buff-aura-range: 16       # バフオーラの範囲（ブロック）
  buff-aura-height: 8       # バフオーラの高さ（ブロック）
  buff-aura-interval: 40    # バフオーラの周期（ティック）
```

> 既存の `slime` セクション設定（`growth-interval`, `max-size`, `earthquake-*`, `enchanted-spawn-chance`, `max-per-server`）はそのまま使用できます。

---

## 📦 アップグレード手順
1. `plugins/RPPacks.jar` を新しい JAR に置き換え
2. サーバーを再起動、または `/rpp reload` を実行
3. `config.yml` に新しい `buff-aura-*` 設定が自動追加されます
